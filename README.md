# Turborepo react-native starter

This is a community-maintained example. If you experience a problem, please submit a pull request with a fix. GitHub Issues will be closed.

## Using this example

Run the following command:

```sh
npx create-turbo@latest -e with-react-native-web
```

## What's inside?

This Turborepo includes the following packages/apps:

### Apps and Packages

- `native`: a [react-native](https://reactnative.dev/) app built with [expo](https://docs.expo.dev/)
- `web`: a [Next.js](https://nextjs.org/) app built with [react-native-web](https://necolas.github.io/react-native-web/)
- `@repo/ui`: a stub [react-native](https://reactnative.dev/) component library shared by both `web` and `native` applications
- `@repo/typescript-config`: `tsconfig.json`s used throughout the monorepo

Each package/app is 100% [TypeScript](https://www.typescriptlang.org/).

### Utilities

This Turborepo has some additional tools already setup for you:

- [Expo](https://docs.expo.dev/) for native development
- [TypeScript](https://www.typescriptlang.org/) for static type checking
- [Prettier](https://prettier.io) for code formatting

📌 Warum ein solches Produkt große Perspektive hat

1️⃣ Sprachbarriere + Rechtsunsicherheit

Polnische Unternehmer in Deutschland kämpfen oft mit:

Deutschlichen Formularen

komplizierten Steuerregeln

Bürokratie

Pflichtmeldungen → Eine polnischsprachige, einfache App schafft echten Nutzen.



---

2️⃣ Komplexe Systeme + schlechte UX

Viele Lösungen (DATEV, ELSTER, Lexware etc.) sind: ❌ nicht intuitiv
❌ unübersichtlich
❌ technisch schwer
→ Für kleine Unternehmer ist das überwältigend.

Eine schlanke, verständliche Oberfläche mit Klartext-Hinweisen wäre ein Riesen-Vorteil.


---

3️⃣ Hohe Konsequenzen bei Fehlern

Ein kleines Versäumnis kann zu: ⚠️ Bußgeldern
⚠️ Steuernachzahlungen
⚠️ Finanzamt-Problemen
→ Viele Unternehmer suchen deshalb Hilfe, Kontrolle & Sicherheit.
